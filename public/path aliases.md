# path Aliases

It is annoying to hardcode paths with slashes and dots, so in Next.js we can do that dynamicly with path aliases.
We create in the root `jsconfig.json` file

```json

{
    "compilerOptions": {
        "baseURL": ".",
        "paths": {
            "@/component/*": ["component/*"],
            "@/config/*": ["config/*"],
            "@/utils/*": ["utils/*"],
            "@/styles/*": ["styles/*"],
            "@/data/*": ["data/*"]
        }
    }

}

```
